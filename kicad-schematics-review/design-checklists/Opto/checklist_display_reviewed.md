# LCD / OLED Display Module Design Checklist

## 1. Display Technology Selection

- [ ] Verify display technology (character LCD, graphic LCD, TFT LCD, OLED, AMOLED, E-Ink, segmented LCD) is appropriate for application
- [ ] Verify resolution meets information density requirements per datasheet
- [ ] Verify color support (monochrome, 16-bit, 24-bit) meets application requirements
- [ ] Verify viewing angle technology (TN, IPS, VA) is adequate for user positions
- [ ] Verify sunlight readability: transflective LCD if used in bright outdoor conditions
- [ ] Verify operating temperature range covers application environment per datasheet
- [ ] Verify power consumption (backlight + controller) meets system power budget

## 2. Interface

### Parallel Interface
- [ ] Verify 8080/6800 parallel interface (8-bit or 16-bit) is compatible with host
- [ ] Verify RGB parallel interface for high-resolution TFT panels — check pixel clock and sync signals
- [ ] Verify data bus routing with matched lengths per datasheet guidelines

### Serial Interface
- [ ] If SPI: verify 3-wire or 4-wire mode is supported by host — verify maximum SPI speed per datasheet
- [ ] If I2C: verify speed is adequate for required refresh rate — check bandwidth against resolution and color depth
- [ ] If DSI (MIPI): verify host has MIPI DSI controller — verify differential pair routing

### LVDS / eDP
- [ ] If LVDS: verify host has LVDS transmitter or external bridge — verify number of differential pairs
- [ ] If eDP: verify host supports eDP interface

## 3. Display Controller

### Integrated Controller (on module)
- [ ] Verify controller supports required resolution and color depth per datasheet
- [ ] Verify controller features: rotation, mirroring, partial update, sleep mode
- [ ] Verify sleep mode current meets system power budget
- [ ] Verify supply voltage is compatible with system rails
- [ ] Verify command set documentation is available

### External Controller (for raw panels)
- [ ] If raw TFT panel without controller: verify external controller (e.g., RA8875, FT81x, SSD1963) or MCU LTDC is used
- [ ] Verify controller supports panel timing parameters (HFP, HBP, VFP, VBP, pixel clock) per panel datasheet

### Frame Buffer
- [ ] Verify frame buffer size is adequate for resolution and color depth
- [ ] If external SDRAM used for frame buffer: verify memory bandwidth meets refresh requirements
- [ ] Verify DMA support for transferring image data to display

## 4. Power Supply

### Display Module Supply
- [ ] Verify VDD logic voltage (1.8V, 2.8V, or 3.3V) per display datasheet
- [ ] If LCD with backlight: verify boost LED driver is used and configured per backlight specification
- [ ] If OLED: verify all required voltages (VCC, VCOM, VLED) per datasheet
- [ ] If TFT: verify panel supplies (AVDD, VGH, VGL) per datasheet — check if generated by integrated charge pump or external DC-DC

### Backlight (LCD)
- [ ] Verify LED backlight configuration (series or parallel) per module specification
- [ ] Verify LED forward voltage and current per datasheet
- [ ] Verify backlight driver (boost converter) meets voltage and current requirements
- [ ] Verify PWM dimming input is available on driver
- [ ] Verify backlight driver efficiency target is adequate
- [ ] Verify backlight power consumption meets system power budget

### Power Sequencing
- [ ] Verify power-up sequence: VDD logic → display panel supplies per datasheet
- [ ] For TFT: verify VGH and VGL are present before gate driver operates per datasheet
- [ ] Verify backlight is enabled after display content is ready
- [ ] Verify power-down sequence: backlight off → display off → VDD off per datasheet

## 5. Backlight (LCD Specific)

### LED Configuration
- [ ] Verify number of LEDs and series/parallel configuration per module specification
- [ ] Verify current per LED string per datasheet
- [ ] Verify dimming method (PWM or analog) per application requirements
- [ ] Verify LED lifetime rating meets product lifetime requirements

### Backlight Driver
- [ ] Verify boost converter topology is appropriate for series LED configuration
- [ ] Verify input voltage range covers system supply variations
- [ ] Verify output voltage capability: N_series × V_f + driver headroom
- [ ] Verify PWM dimming frequency > 200 Hz to avoid visible flicker
- [ ] For camera applications: verify no beat frequencies with frame rate

## 6. Display Glass / Physical

- [ ] Verify active area dimensions fit enclosure window
- [ ] Verify module footprint fits PCB layout
- [ ] Verify mounting method (screw holes, adhesive frame, bezel) is compatible with enclosure
- [ ] If touch panel integrated: verify capacitive or resistive type meets application requirements
- [ ] If cover glass used: verify anti-glare or anti-reflective coating requirements
- [ ] Verify connector type (ZIF flex cable, pin header, solderable pads) is compatible with PCB
- [ ] Verify flex cable strain relief and folding direction per mechanical constraints

## 7. Touch Interface (if integrated)

- [ ] If resistive touch: verify SPI interface and controller (e.g., ADS7846, TSC2046) — verify calibration requirements
- [ ] If capacitive touch: verify I2C interface and controller — verify multi-touch and gesture support per datasheet
- [ ] Verify touch controller I2C address configuration
- [ ] Verify touch panel sensitivity calibration per datasheet
- [ ] If outdoor/wet use: verify water rejection capability

## 8. Electrostatic Discharge (ESD)

- [ ] Verify TVS diodes on all exposed signal lines (data, touch, backlight)
- [ ] Verify EMI filter / ferrite bead on flex cable
- [ ] Verify display frame/chassis is connected to system GND
- [ ] If exposed: verify conductive gasket or spring contact between display and enclosure
